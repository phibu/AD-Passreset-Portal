<objective>
Fix the `MissingEndCurlyBrace` ParserError that currently prevents `deploy/Uninstall-PassReset.ps1`
from executing. Root cause per CONTEXT (D-Claude's discretion on STAB-005) and PATTERNS.md §STAB-005:
a combination of (a) file encoding lacking a BOM and (b) Unicode `─` (U+2500) box-drawing characters
in section dividers confuses Windows PowerShell 5.1's default-encoding parser. Fix with the
lowest-risk change: re-save as UTF-8 with BOM and replace U+2500 with ASCII `---`.

Purpose: Closes ROADMAP.md Phase 7 Success Criterion #5 (gh#39). Operators on fresh Windows
Server 2019 boxes running stock Windows PowerShell 5.1 currently cannot uninstall.

Output: `Uninstall-PassReset.ps1` parses and executes cleanly on both PS 5.1 and PS 7.x.
Defensive audit report (inline) confirms whether `Install-PassReset.ps1` has the same issue.
</objective>

<task type="auto">
  <name>Task 1: Replace U+2500 dividers with ASCII and re-save Uninstall-PassReset.ps1 as UTF-8 with BOM</name>
  <files>deploy/Uninstall-PassReset.ps1</files>
  <read_first>
    - deploy/Uninstall-PassReset.ps1 (full file — see current divider positions)
    - .planning/phases/07-installer-deployment-fixes/07-PATTERNS.md §STAB-005 (lines 137-153)
    - .planning/phases/07-installer-deployment-fixes/07-CONTEXT.md §STAB-005 (line 35)
  </read_first>
  <action>
    1. Read `deploy/Uninstall-PassReset.ps1` verbatim.
    2. Replace EVERY occurrence of the character `─` (U+2500 BOX DRAWINGS LIGHT HORIZONTAL) with `-`
       (ASCII hyphen-minus, U+002D). Preserve the visual length of each divider line —
       if the original used 77 `─` characters, the replacement uses 77 `-` characters.
       Apply to section comment blocks only (lines starting with `#`); do NOT alter any
       user-facing `Write-Host` message content.
    3. DO NOT restructure, rename, reformat, or re-order anything else. No refactoring. No comment
       rewording. Only the character substitution above.
    4. Write the file back using UTF-8 **with BOM** encoding. In PowerShell this is
       `[System.IO.File]::WriteAllText($path, $content, [System.Text.UTF8Encoding]::new($true))`.
       Do NOT use `Set-Content -Encoding UTF8` on PS 5.1 (that writes UTF-8 WITHOUT BOM on 5.1,
       defeating the fix). If the editor tooling defaults to BOM-less UTF-8, use the .NET API
       call above or `Out-File -Encoding utf8BOM` on PS 7.x.
    5. Verify the first three bytes of the file are 0xEF 0xBB 0xBF via
       `[byte[]](Get-Content $path -Encoding Byte -TotalCount 3)` (PS 5.1) or
       `[byte[]](Get-Content $path -AsByteStream -TotalCount 3)` (PS 7.x).
    6. Verify the file contains zero U+2500 characters via
       `(Get-Content $path -Raw) -match '[\u2500]'` returning `$False`.
  </action>
  <verify>
    <automated>
      powershell.exe -NoProfile -Command "
        $p = 'deploy/Uninstall-PassReset.ps1';
        $bytes = [byte[]](Get-Content $p -Encoding Byte -TotalCount 3);
        if ($bytes[0] -ne 0xEF -or $bytes[1] -ne 0xBB -or $bytes[2] -ne 0xBF) { throw 'Missing UTF-8 BOM' };
        if ((Get-Content $p -Raw) -match '[\u2500]') { throw 'Still contains U+2500' };
        Write-Host 'OK'
      "
    </automated>
  </verify>
  <acceptance_criteria>
    - `[byte[]](Get-Content deploy/Uninstall-PassReset.ps1 -Encoding Byte -TotalCount 3)` returns `0xEF, 0xBB, 0xBF`
    - `(Get-Content deploy/Uninstall-PassReset.ps1 -Raw) -match '[\u2500]'` returns `$False`
    - `(Get-Content deploy/Uninstall-PassReset.ps1 -Raw).Length` differs from the pre-change length by
      exactly the number of U+2500 characters removed (each was 3 bytes UTF-8 and is now 1 byte ASCII —
      so `-2 * count` bytes; character count is unchanged)
    - `git diff deploy/Uninstall-PassReset.ps1` shows ONLY divider-character replacements
      (no renamed variables, no restructured logic, no changed strings inside Write-Host calls)
  </acceptance_criteria>
  <done>Uninstall-PassReset.ps1 is UTF-8 with BOM, contains no U+2500 characters, and git diff shows only cosmetic divider changes.</done>
