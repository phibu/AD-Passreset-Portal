<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/health | Anonymous HTTP(S) request — response body is public-visible to load balancers and anyone who can reach the endpoint |
| HealthController → AD/SMTP | Outbound TCP from IIS worker thread to domain controller and SMTP relay |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Information Disclosure | HealthController response body | mitigate | D-04: response contains only status enums + latency integers + timestamps. SMTP host/port are already configured values, not secrets. Test 4 asserts `SmtpSettings.Password`, reCAPTCHA PrivateKey, and LDAP password strings are NOT present in the raw response body. |
| T-10-01-02 | Denial of Service | SMTP TCP probe blocking IIS worker thread | mitigate | D-05 + R-01: `TcpClient.ConnectAsync(host, port, CancellationTokenSource(3s).Token)` hard deadline. `SmtpProbeTests` asserts blackhole-host connect cancels ≤ 4s. No sync `Connect()` remains. |
| T-10-01-03 | Information Disclosure | Error details in exception messages | mitigate | D-04: exceptions caught inside `CheckSmtpAsync`/`CheckAdConnectivityAsync` are logged via `_logger.LogWarning` (server-side only). Response returns stable "unhealthy" string; no exception message leaks. |
| T-10-01-04 | Tampering | Load balancer interprets `status` field | mitigate | Contract preserved: aggregate `status` remains `"healthy"|"degraded"|"unhealthy"` at the same top-level key. Pitfall 1 guarded — nested `checks.ad` shape change is additive diagnostic only. |
| T-10-01-05 | Denial of Service | No rate limit on /api/health | accept | Endpoint is already anonymous and unauthenticated by design (load balancer probe). Adding rate limit would defeat its purpose. Aggregate cost per request is bounded by 3s SMTP + 3s AD timeouts = ≤ 6s worst case. |
</threat_model>
