    1. Add new function `Test-HttpsBinding` near the Phase 7 helper block (around line 128, after the `Abort` function definition and before the first `Get-` helper like `Get-SchemaKeyManifest`):
       ```powershell
       function Test-HttpsBinding {
           <#
           .SYNOPSIS
             STAB-016 (D-13): Warn when the target IIS site has no HTTPS binding.
           .DESCRIPTION
             Warn-not-block helper invoked AFTER New-Website / cert-bind steps. Does NOT
             call Abort — operators running offline staging may legitimately have no HTTPS
             binding, and TLS-terminating reverse-proxy deployments front the site on HTTP.
             Reverse-proxy setups should also flip WebSettings.EnableHttpsRedirect=false.
           #>
           [CmdletBinding()]
           param(
               [Parameter(Mandatory)]
               [string]$SiteName
           )

           $httpsBindings = @(Get-WebBinding -Name $SiteName -Protocol https -ErrorAction SilentlyContinue)
           if ($httpsBindings.Count -eq 0) {
               Write-Warn "Site '$SiteName' has no HTTPS binding. PassReset requires TLS in production."
               Write-Warn "  - Re-run Install-PassReset.ps1 with -CertThumbprint <thumbprint> to configure HTTPS, or"
               Write-Warn "  - Add an HTTPS binding manually via IIS Manager before exposing the site."
               Write-Warn "  - For TLS-terminating reverse-proxy deployments, set WebSettings.EnableHttpsRedirect=false in appsettings.Production.json."
           } else {
               Write-Ok "HTTPS binding verified on site '$SiteName' ($($httpsBindings.Count) binding(s))"
           }
       }
       ```

       - Use `Write-Warn` / `Write-Ok` — do not `Write-Host` directly.
