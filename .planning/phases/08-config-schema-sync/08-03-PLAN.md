<objective>
Wire fail-fast startup validation for every options class per D-05/D-07/D-08. Add `IValidateOptions<T>` validators in `src/PassReset.Web/Models/` (one per options class), refactor `Program.cs` to `AddOptions<T>().Bind(...).ValidateOnStart()`, and add a try/catch that writes `OptionsValidationException` to the Windows Application Event Log under source `PassReset` before the host crashes (so operators see the failure in Event Viewer, not just IIS 502).

Output: 6 new validator classes + 1 updated existing validator + Program.cs refactor + Event Log write path.

**Decision (planner authority):** use direct `EventLog.WriteEntry("PassReset", ..., EventLogEntryType.Error)` — vendor-conservative (no NuGet add), and `System.Diagnostics.EventLog` is already available in `net10.0-windows`. Event Log source registration is owned by the installer (Install-PassReset.ps1 — handled in plan 08-04).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| appsettings.Production.json → ASP.NET Core process | Untrusted config crosses; validators are the gate. |
| process → Windows Application Event Log | Diagnostic write; must NOT include secrets per D-08 information-disclosure mitigation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07 | Information disclosure | Validation failure messages (could leak secrets) | mitigate | Validators redact secret-field values in `(got "...")` segment. Tests assert no secret value appears in any Fail message. |
| T-08-08 | Information disclosure | Event Log entry contents | mitigate | Event Log message is built from `ex.Failures` — same redacted strings produced by validators. No raw config dump. |
| T-08-09 | Tampering | EventLog.SourceExists check | accept | Source registration is owned by installer (plan 08-04). If unregistered, write is silently swallowed; exception still re-throws and surfaces via IIS stdout logging. |
| T-08-10 | Denial of service | Startup throws repeatedly on bad config | accept | This is the desired fail-fast behavior per D-07. Operator must edit config + restart IIS. Documented in plan 08-08 docs. |
| T-08-11 | Elevation of privilege | EventLog.WriteEntry on misconfigured machine | accept | EventLog.WriteEntry runs under app pool identity. Failure to write (permissions) is swallowed; original exception still propagates. |
</threat_model>
