<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → installer params | `-ConfigSync` value crosses; `[ValidateSet]` enforces allowed values. |
| live appsettings.Production.json → installer | Pre-flight validates before any sync mutates the file. |
| installer → Windows Event Log | New-EventLog requires admin; installer already runs as admin (`#Requires -RunAsAdministrator`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-12 | Tampering | -ConfigSync param injection | mitigate | `[ValidateSet('Merge','Review','None')]` PowerShell-native enum validation. Any other value rejected at param binding before script body runs. |
| T-08-13 | Elevation of privilege | New-EventLog source registration | accept | Requires admin; installer already enforces `#Requires -RunAsAdministrator`. Source registration is one-time and idempotent (`SourceExists` check). |
| T-08-14 | Denial of service | Pre-flight Abort on minor schema drift | mitigate | Schema is conservative on `required:` (only must-exist keys per Task 1 of plan 08-01). Operators editing extras (`additionalProperties: true` everywhere) won't trip it. |
| T-08-15 | Information disclosure | Test-Json error message echoed by Abort | mitigate | Test-Json reports field path + reason; never includes value. Schema has no `default:` for secret fields (passwords) so the schema doesn't carry secrets either. |
| T-08-16 | Pre-flight bypass | Operator could try to skip via removing schema | mitigate | If schema is missing, only Write-Warn + continue (graceful) — operator gets visible warning. CI in 08-02 prevents shipping a release without schema. |
</threat_model>
