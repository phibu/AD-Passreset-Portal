## Phase 07 — v1.3.1 AD Diagnostics

**Goal:** Diagnose intermittent `0x80070005 (E_ACCESSDENIED)` and related password-change failures by adding structured logging around every step of the AD password-change flow. External behavior unchanged.

**Plans:**
- ✅ 07-01 — 8 tasks covering middleware + helper + provider refactor + decorator + controller + tests

**Success criteria — all met:**
1. Every AD password-change path logs structured events for user lookup, `ChangePasswordInternal`, and `Save` (step-before/after at `Debug` level with `ElapsedMs`).
2. `DirectoryServicesCOMException` and `PasswordException` emit structured `ExceptionChain` arrays (type, HResult as `0x{X8}`, message, depth).
3. Targeted catches exist for all three exception types: `PasswordException` (chain walker), `PrincipalOperationException` (plain targeted, default destructure), `DirectoryServicesCOMException` (chain walker).
4. Every request correlates via W3C `Activity.TraceId` + `SpanId`, pushed once by `TraceIdEnricherMiddleware`.
5. Lockout decorator emits `Debug` events for counter increments and eviction sweeps; existing `Warning` logs for `ApproachingLockout` / `PortalLockout` preserved.
6. No passwords or plaintext ever appear in log output — verified by `PasswordLogRedactionTests` across provider, decorator, and end-to-end controller path.
7. User-facing error responses unchanged from v1.3.0.

## Key Decisions

- **W3C `Activity.TraceId`** over `HttpContext.TraceIdentifier` — zero infra cost, forward-compatible with OpenTelemetry if Phase 4 later adds distributed telemetry.
- **Step events at `Debug` level** (not `Information`) — production stays quiet; operators flip `Serilog:MinimumLevel:Default` to diagnose.
- **Exception-chain walker scoped to COM + PasswordException only** — these are the two types where HResult + depth genuinely helps root-cause E_ACCESSDENIED. `PrincipalOperationException` gets a plain targeted catch with default Serilog destructure.
- **AD-context scope opened only inside `if (userPrincipal != null)` branch** — avoids `PrincipalContext.ConnectedServer` null pre-bind (RESEARCH pitfall 3).
- **Handwritten `ListLogEventSink`** for redaction tests — zero new Serilog packages, fits existing test style.
- **No new config keys** — diagnostics are always-on at their configured level; operators use the existing Serilog `MinimumLevel` knob.
- **Resolved CONTEXT/ROADMAP conflict** in favor of option (b): walker limited to 2 exception types; `PrincipalOperationException` still gets distinct log context via its own targeted catch.

## Issues Hit During Close

- **Executor turn-limit:** The initial executor ran out of turns while debugging the final redaction test. Resumed via a second executor call.
- **Serilog test-wiring gotcha:** `builder.Host.UseSerilog(...)` in `Program.cs` registers the Serilog `ILoggerFactory` in a way that bypasses standard `ILoggerProvider` chains — the initial `ConfigureTestServices(AddLogging + AddProvider)` override was silently ignored and the capture sink received zero events. Fixed by replacing the `ILoggerFactory` service directly with a `SerilogLoggerFactory` bound to the capture sink (last registration wins). Test-only change; no product code touched.
