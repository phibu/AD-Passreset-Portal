## Bugs

- [x] **BUG-004**: Structured diagnostic logging around every step of the AD password-change flow. Every request correlated via W3C `Activity.TraceId` (pushed by new `TraceIdEnricherMiddleware`). Request-scoped `BeginScope` in `PasswordController.PostAsync` with `Username` / `TraceId` / `ClientIp`. Nested AD-context scope in `PasswordChangeProvider.PerformPasswordChangeAsync` (`Domain`, `DomainController`, `IdentityType`, `UserCannotChangePassword`, `LastPasswordSetUtc`) opened only after successful user lookup. Step-before/after `Debug` events around user lookup, `ChangePasswordInternal`, `Save` (with `ElapsedMs`). Custom `ExceptionChainLogger` emits structured `ExceptionChain` array for `DirectoryServicesCOMException` and `PasswordException`; `PrincipalOperationException` gets plain targeted catch with default destructure. `LockoutPasswordChangeProvider` adds `Debug` events for counter increments and eviction sweeps; existing `Warning` logs preserved. No passwords/plaintext written to logs — enforced by `PasswordLogRedactionTests` sentinel assertions across provider, decorator, and end-to-end controller path. No new config keys. User-facing error responses unchanged from v1.3.0.
