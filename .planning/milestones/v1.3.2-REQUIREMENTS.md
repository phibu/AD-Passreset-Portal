# PassReset — v1.3.2 Requirements (ARCHIVED)

**Status:** ✅ Code review fix rollup delivered
**Shipped:** 2026-04-16 · tag `v1.3.2`

## Scope

Patch release rolling up the post-v1.3.1 deep code review findings (WR-01, WR-02, WR-03) from the phase 07 deep re-review. No new requirements, no behavior changes — internal hardening of the diagnostic logging paths introduced in v1.3.1.

## Findings Addressed

| ID | Finding | Fix |
|---|---|---|
| WR-01 | `ExceptionChainLogger` had no cycle protection or depth bound; adversarial/malformed `InnerException` chains could spin indefinitely and bloat a single log event | Added cycle protection (visited-set) + configurable depth bound in `ExceptionChainLogger.cs` |
| WR-01 | `PasswordChangeProvider` COM error log template arity did not match named-argument list, causing Serilog to drop the `ExceptionChain` property on the COM path | Aligned template placeholder count with named args in `PasswordChangeProvider.cs` |
| WR-02 | `PasswordLogRedactionTests` test names were misleading — they claimed to assert provider catch-path behavior but actually exercised a fake provider | Renamed tests to reflect what they assert; added direct `ExceptionChainLogger` sentinel test for AD-message passthrough (documented accepted-risk) |
| WR-03 | `LockoutPasswordChangeProvider.IncrementCounter` emitted a redundant `Debug` log; outer call site already emits `Warning` with counter state | Removed the redundant `LogDebug` — no loss of operator signal |
