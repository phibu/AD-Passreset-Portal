# Secret Management

PassReset handles several sensitive configuration values that must be protected in production deployments.

## Secrets Inventory

| Secret | Configuration Path | Used By |
|--------|--------------------|---------|
| LDAP bind password | `PasswordChangeOptions:LdapPassword` | AD password change provider |
| SMTP relay password | `SmtpSettings:Password` | Email notification service |
| reCAPTCHA private key | `ClientSettings:Recaptcha:PrivateKey` | Server-side CAPTCHA verification |

## Default Behavior

By default, secrets are stored in `appsettings.Production.json` on the IIS server. This file is:
- Generated by `Install-PassReset.ps1` on first install with empty placeholder values
- Excluded from version control via `.gitignore`
- Readable only by the IIS app pool identity and local administrators (NTFS ACLs set by the installer)

## Hardening Options

### Option 1: Environment Variables (Recommended)

ASP.NET Core reads environment variables using the `__` (double underscore) separator. Set these on the IIS application pool or via system environment variables:

```
PasswordChangeOptions__LdapPassword=YourLdapPassword
SmtpSettings__Password=YourSmtpPassword
ClientSettings__Recaptcha__PrivateKey=YourRecaptchaKey
```

Environment variables take precedence over `appsettings.json` values. This avoids storing secrets on disk entirely.

### Option 2: Windows DPAPI (Advanced)

For servers that cannot use environment variables, consider encrypting the `appsettings.Production.json` file using Windows Data Protection API. The ASP.NET Core Data Protection system can be configured to use DPAPI for key protection.

### Option 3: Skip LDAP Credentials Entirely

When `UseAutomaticContext: true` (the default for domain-joined servers), no LDAP username or password is needed. The application uses the computer account's domain context, which is the most secure option as no credentials are stored anywhere.

## What Is NOT a Secret

The following values are safe to include in configuration files:
- `ClientSettings:Recaptcha:SiteKey` — this is the public site key, visible in the browser
- `PasswordChangeOptions:DefaultDomain` — the AD domain name
- `PasswordChangeOptions:LdapHostnames` — the LDAP server hostnames
- `SmtpSettings:Host` — the SMTP relay hostname

## Security Controls

- `Recaptcha.PrivateKey` is marked `[JsonIgnore]` on the `ClientSettings` model, preventing it from being returned by `GET /api/password`
- `appsettings.Production.json` and `secrets.json` are in `.gitignore`
- The installer sets NTFS permissions restricting file access to the app pool identity
